int UART_read ( UARTbus  bus,
unsigned char *  data,
unsigned int  size 
)

Performs a blocking read transfer by sleeping the task until the specified transfer is complete. This allows other tasks to execute while the transfer is made using DMA but the calling task is blocked.

Parameters
busThe UART bus over which the transfer should be made.
dataMemory location where the received data should be stored.
sizeNumber of bytes to be received.
Returns
-3 if creating a semaphore for the transfer fails, -2 if input parameters are invalid or the driver is not initialized, -1 if adding the transfer to the queue fails, 0 on success. 2 for general errors during the transfer, 4 for a framing error, 8 for a parity error, 12 if both framing and parity errors occur during the transfer, 16 if an overrun error occurs.
Note
If a negative error is returned, the transfer was never attempted (for example due to an invalid input). If a positive error is returned, an error occurred during the transfer.
The positive error values correspond to the members of UARTtransferStatus enumerated data type.
int UART_setPostTransferDelay ( UARTbus  bus,
portTickType  delay 
)

Post transfer delay is used to add a delay between successive transactions on the bus. In case of UART_queueTransfer, this value is passed as a part of the transfer specifier and the value specified here is superseded by the value specified in the transfer specifier. In case of all other functions (UART_writeRead, UART_write, UART_read) the driver will use the value set using this function.

Note
If no value is set using this function, a value of zero is used by the driver.
Parameters
busThe bus for which this delay should be set.
delayNumber of ticks the driver should wait between transactions.
Returns
-1 if the specified bus is not valid.
